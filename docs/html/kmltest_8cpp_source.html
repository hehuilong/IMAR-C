00033 //----------------------------------------------------------------------
00034 // kmltest
00035 //
00036 // This program is a driver for testing and evaluating various algorithms
00037 // for the k-means problem, for point clustering in multi-dimensional
00038 // spaces.  It allows the user to generate or input data sets, to specify
00039 // the number of centers and generate or input their initial positions,
00040 // and then to run one of a number of k-means procedures.
00041 //
00042 // Overview:
00043 // ---------
00044 // The test program is run as follows:
00045 // 
00046 //      kmltest < test_input > test_output
00047 //
00048 // where the test_input file contains a list of directives as described
00049 // below.  Directives consist of a directive name, followed by list of
00050 // arguments (depending on the directive).  Arguments and directives are
00051 // separated by white space (blank, tab, and newline).  String arguments
00052 // are not quoted, and consist of a string of nonwhite chacters.  A
00053 // character "#" denotes a comment.  The following characters up to
00054 // the end of line are ignored.  Comments may only be inserted between
00055 // directives (not within the argument list of a directive).
00056 //
00057 // Algorithm Overview:
00058 // -------------------
00059 // The generic algorithm begins by generating an initial solution curr
00060 // and saving it in best.  These objects are local to the KMlocal
00061 // structure.   The value of curr reflects the current solution and
00062 // best reflects the best solution seen so far.
00063 //
00064 // The generic algorithm consists of some number of basic iterations,
00065 // called stages.  Each stage involves the execution of one step of
00066 // either the swap heuristic or Lloyd's algorithm.  Each of the
00067 // algorithms differ in how they apply these stages.
00068 //
00069 // Stages are grouped into runs.  Intuitively, a run involves a
00070 // (small) number of stages in search of a better solution.  A run might
00071 // end, say, because a better solution was found or a fixed number of
00072 // stages have been performed without any improvement. 
00073 //
00074 // After a run is finished, we check to see whether we want to accept
00075 // the solution.  Presumably this happens if the cost is lower, but it
00076 // may happen even if the cost is inferior in other circumstances (e.g.,
00077 // as part of a simulated annealing approach).  Accepting a solution
00078 // means copying the current solution to the saved solution.  In some
00079 // cases, the acceptance may involve reseting the current solution to a
00080 // random solution.
00081 //
00082 // Algorithm Details:
00083 // ------------------
00084 // There are some concepts that are important to run/phase transitions.
00085 // One is the maximum number of stages.  Most algorithms provide some
00086 // sort of parameter that limits the number of stages that the algorithm
00087 // can spend in a particular run (before giving up).  The second is the
00088 // relative distortion loss, or RDL. (See also KMterm.h.) The RDL is
00089 // defined:
00090 //
00091 //              initDistortion - currDistortion
00092 //       RDL =  -------------------------------
00093 //                    initDistortion
00094 //
00095 // Note that a positive value indicates that the distortion has
00096 // decreased.  The definition of "initDistortion" depends on the
00097 // algorithm.  It may be the distortion of the previous stage (RDL =
00098 // consecutive RDL), or it may be the distortion at the beginning of the
00099 // run (RDL = accumulated RDL).
00100 //
00101 // Here is a more detailed explanation of the algorithms.
00102 //
00103 // KMlocalLloyds
00104 // -------------
00105 //      This is Lloyd's algorithm with random restarts The algorithm is
00106 //      broken into phases, and each phase is broken into runs.  Each
00107 //      phase starts by sampling center points at random.  Each run is
00108 //      provided two parameters, a maximum number of runs per stage
00109 //      (max_run_stage) and a minimum accumulated relative distortion
00110 //      loss (min_accum_rdl).  If the accumulated RDL for the run
00111 //      exceeds this value, then the run ends in success.  If the number
00112 //      of stages is exceeded before this happens, the run ends in
00113 //      failure.  The phase ends with the first failed run.
00114 // KMlocalSwap
00115 // -----------
00116 //      This algorithm iteratively changes centers by performing swaps.
00117 //      Each run consists of a given number (max_swaps) executions of
00118 //      the swap heuristic.
00119 // KMlocalEZ_Hybrid
00120 // ----------------
00121 //      This implements a simple hybrid algorithm (compared to
00122 //      KMlocalHybrid).  The algorithm performs only one swap, followed
00123 //      by some number of iterations of Lloyd's algorithm.  Lloyd's
00124 //      algorithm is repeated until the consecutive RDL falls below a
00125 //      given threshold.
00126 //
00127 //      A stage constitutes one invocation of the Swap or Lloyd's
00128 //      algorithm.  A run consists of a single swap followed by a
00129 //      consecutive sequence of Lloyd's steps.  A graphical
00130 //      representation of one run is presented below.  The decision to
00131 //      make another pass through Lloyd's is based on whether the
00132 //      relative improvement since the last stage (consecutive relative
00133 //      distortion loss) is above a certain fixed threshold
00134 //      (min_consec_rdl).
00135 // KMlocalHybrid
00136 // -------------
00137 //      This implements a more complex hybrid algorithm, which combines
00138 //      both of swapping and Lloyd's algorithm with a variant of
00139 //      simulated annealing.  The algorithm's execution is broken into
00140 //      the following different processes: one swap, a consecutive
00141 //      sequence of Lloyd's steps, and an acceptance test.  If we pass
00142 //      the acceptance test, we take the resulting solution, and
00143 //      otherwise we restore the old solution.
00144 //
00145 //      The decision to perform another Lloyd's step or go on to
00146 //      acceptance is based on whether the relative improvement since
00147 //      the last stage (consecutive relative distortion loss) is above a
00148 //      certain fixed threshold (min_consec_rdl).
00149 //
00150 //      If the resulting solution is better than the saved solution,
00151 //      then we accept it.  Otherwise, we use the simulated annealing
00152 //      decision choice (described below) to decide whether to accept
00153 //      it.
00154 //
00155 //      Simulated Annealing Choice
00156 //      --------------------------
00157 //      The choice to accept a poorer solution occurs with probability
00158 //
00159 //              exp(RDL/T),
00160 //
00161 //      where RDL is the relative distortion loss (relative to the
00162 //      saved solution), and T is the current temperature.  Note that
00163 //      if RDL > 0 (improvement) then this quantity is > 1, and so we
00164 //      always accept.
00165 //
00166 //      The temperature value T is a decreasing function of the number
00167 //      of the number of stages.  It starts at some initial value T0 and
00168 //      decreases slowly over time.  Rather than using the standard (slow)
00169 //      Boltzman annealing schedule, we use the following fast annealing
00170 //      schedule, every L stages we set
00171 //
00172 //              T = TF * T,
00173 //
00174 //      where:
00175 //              L (= temp_run_length) is an integer parameter set by the
00176 //                  user.  (Presumably it depends on the number of
00177 //                  centers and the dimension of the space.)
00178 //              TF (= temp_reduc_factor) is a real number of the form 1-x,
00179 //                  for some small x.
00180 //
00181 //      The initial temperature T0 is a tricky thing to set.  The user
00182 //      supplies a parameter p0 = init_prob_accept, the initial
00183 //      probability of accepting a random swap.  However, the
00184 //      probability of acceting a swap depends on the given RDL value.
00185 //      To estimate this, for the first L runs we use p0 as the
00186 //      probability.  Over these runs we compute the average rdl value.
00187 //      Once the first L runs have completed, we set T0 so that:
00188 //
00189 //              exp(-averageRDL/T0) = initProbAccept,
00190 //
00191 //      or equivalently
00192 //
00193 //              T0 = -averageRDL/(ln initProbAccept).
00194 //
00195 // Basic operations:
00196 // -----------------
00197 // The test program can perform the following operations.  How these
00198 // operations are performed depends on the options which are described
00199 // later. 
00200 //
00201 //      Data Generation:
00202 //      ----------------
00203 //      read_data_pts <file>    Create a set of data points whose
00204 //                              coordinates are input from file <file>.
00205 //                              Prior to this, data_size must be set.
00206 //                              At most data_size points will be read
00207 //                              from the file.  The actual number of
00208 //                              points in the file may be less.
00209 //      gen_data_pts            Create a set of data points whose
00210 //                              coordinates are generated from the
00211 //                              current point distribution.
00212 //
00213 //      Running k-means:
00214 //      ----------------
00215 //      run_kmeans <string>     Apply k-means clustering to the current
00216 //                              point set and clusters.  The string
00217 //                              specifies the desired version of k-means.
00218 //                              These include:
00219 //                                  lloyd = runs Lloyd's algorithm using
00220 //                                          the filtering algorithm.
00221 //                                  swap  = runs the swap heuristic.
00222 //                                  hybrid = runs Lloyd's algorithm
00223 //                                          and the swap heuristic in
00224 //                                          alternating steps.
00225 //                                  EZ-hybrid = a simpler version of
00226 //                                          hybrid. One swap followed
00227 //                                          by some number of Lloyd's.
00228 //
00229 //      Miscellaneous: (Strings may have no embedded blanks.)
00230 //      -----------------------------------------------------
00231 //      title <string>          Experiment title.
00232 //      print <string>          Output string to console (cerr).
00233 //      get_distortion          Computes and prints distortion (the sum
00234 //                              of squared distances for the current
00235 //                              centers).  Note that the kmeans algorithms
00236 //                              compute and print the distortion for all
00237 //                              stages but stage 0, if the stats level
00238 //                              is set to stage.
00239 //
00240 // Options:
00241 // --------
00242 // How these operations are performed depends on a set of options.
00243 // If an option is not specified, a default value is used. An option
00244 // retains its value until it is set again.  String inputs are not
00245 // enclosed in quotes, and must contain no embedded white space (sorry,
00246 // this is C++'s convention).
00247 //
00248 // Options affecting data and query point generation:
00249 // --------------------------------------------------
00250 //      kcenters <int>          Number of centers.  Default = 5.
00251 //      dim <int>               Dimension of space.
00252 //      seed <int>              Seed for random number generation.
00253 //      data_size <int>         Number of data points to generate for
00254 //                              gen_data_pts points and the maximum
00255 //                              number of data points to be read for
00256 //                              read_data_pts.  If this exceeds
00257 //                              max_data_size, then max_data_size is
00258 //                              incremented to match this value.
00259 //                              Default = 100.
00260 //      std_dev <float>         Standard deviation (used in gauss, and
00261 //                              clustered distributions).  This is the
00262 //                              "small" distribution for clus_ellipsoids.
00263 //                              Default = 1.
00264 //      std_dev_lo <float>      Low and high standard deviations (used in
00265 //      std_dev_hi <float>      clus_ellipsoids).  Default = 1.
00266 //      corr_coef <float>       Correlation coefficient (used in co-gauss
00267 //                              and co_lapace distributions). Default = 0.05.
00268 //      colors <int>            Number of color classes (clusters) (used
00269 //                              in the clustered distributions).  Def. = 5.
00270 //      new_clust               Once generated, cluster centers are not
00271 //                              normally regenerated.  This is so that both
00272 //                              centers and data points can be generated
00273 //                              using the same set of clusters.  This option
00274 //                              forces new cluster centers to be generated
00275 //                              with the next generation of either data or
00276 //                              center points.
00277 //      max_clus_dim <int>      Maximum dimension of clusters (used in
00278 //                              clus_orth_flats and clus_ellipsoids).
00279 //                              Default = 1.
00280 //      distribution <string>   Type of input distribution
00281 //                                  uniform     = uniform over cube [-1,1]^d.
00282 //                                  gauss       = Gaussian with mean 0
00283 //                                  laplace     = Laplacian, mean 0 and var 1
00284 //                                  co_gauss    = correlated Gaussian
00285 //                                  co_laplace  = correlated Laplacian
00286 //                                  clus_gauss  = clustered Gaussian
00287 //                                  clus_orth_flats = clusters of orth flats
00288 //                                  clus_ellipsoids = clusters of ellipsoids
00289 //                                  multi_clus  = multi-sized clusters
00290 //                              See the file rand.cc for further information.
00291 //
00292 // Options affection general program behavior:
00293 // -------------------------------------------
00294 //      stats <string>          Level of statistics output
00295 //                                  silent       = no output,
00296 //                                  exec_time   += execution time only
00297 //                                  summary     += summary of complete kmeans
00298 //                                  stage       += summary of each stage
00299 //                                  step        += show each step
00300 //                                  centers     += print centers each time
00301 //                                  tree        += show the tree.
00302 //                              Default = "summary".
00303 //      print_points <string>   Print points after reading/generating?
00304 //                              Argument is either "yes" or "no".
00305 //                              Default = "no".
00306 //      show_assignments <string>
00307 //                              Print the indices of the center to
00308 //                              which each point has been assigned along
00309 //                              with its distance to this center, after
00310 //                              the algorithm terminates.  Argument is
00311 //                              either "yes" or "no".
00312 //                              Default = "no".
00313 //      validate <string>       Test (through a slow brute-force search)
00314 //                              that the center assignments requested by
00315 //                              show_assignments is correct.  This is
00316 //                              used for debugging, and will slow
00317 //                              overall execution.  Argument is either
00318 //                              "yes" or "no".
00319 //                              Default = "no".
00320 //                              (Note: There are other elements of the
00321 //                              computation that could be validated.
00322 //                              Perhaps with time these will be included
00323 //                              under this option.)
00324 //
00325 // Options affecting termination:
00326 // ------------------------------
00327 // The way of controlling the program's termination is to specify the
00328 // maximum number of stages.  (In theory, a better way would be to
00329 // determine when the algorithm has converged, but this seems to be a
00330 // very complex task to me.)  Each time the algorithm moves the center
00331 // points and recomputes the distortion constitutes a stage.  The
00332 // maximum number of stages is based on the number of data points n
00333 // (data_size) and the number of centers k (kcenters) and four
00334 // coefficients, a,...,d, using the following formula:
00335 //
00336 //      MAX_STAGE = a + (b*k + c*n)^d
00337 //
00338 // In addition there are the following parameters.  They are based
00339 // on the "relative distortion loss" (RDL), which is defined to be
00340 // the relative decrease in the distortion.
00341 //
00342 //      max_tot_stage 4*<float>
00343 //                              Maximum total stages given as paramters
00344 //                              (a,...,d).
00345 //                              Default: 0,0,0,0.
00346 //
00347 // Options affecting algorithm execution:
00348 // --------------------------------------
00349 //
00350 //   Options used in Lloyd's Algorithm and Hybrid Algorithms:
00351 //   --------------------------------------------------------
00352 //      damp_factor <float>     A dampening factor in the interval
00353 //                              (0,1].  The value 1 is the standard
00354 //                              Lloyd's algorithm.  Othewise, each point
00355 //                              is only moved by this fraction of the
00356 //                              way from its current location to the
00357 //                              centroid.  Default: 1
00358 //      min_accum_rdl <float>   This is used in Lloyd's algorithm
00359 //                              algorithm which perform multiple swaps.
00360 //                              When performing p swaps, we actually may
00361 //                              perform fewer than p.  We stop
00362 //                              performing swaps, whenever the total
00363 //                              distortion (from the start of the run)
00364 //                              has decreased by at least this amount.
00365 //                              Default: 0.10
00366 //      max_run_stage <int>     This is used in Lloyd's algorithm.  A
00367 //                              run terminates after this many stages.
00368 //                              Default: 100
00369 //
00370 //   Options specific to the swap algorithm:
00371 //   ---------------------------------------
00372 //      max_swaps <int>         Maximum swaps at any given stage.
00373 //                              Default: 1
00374 //
00375 //   Options specific to the hybrid algorithms:
00376 //   ------------------------------------------
00377 //      min_consec_rdl <float>  This is used in the hybrid algorithms.
00378 //                              If the RDL of two consecutive runs is
00379 //                              less than this value Lloyd's algorithm
00380 //                              is deemed to have converged, and the run
00381 //                              ends.
00382 //   Options specific to the (complex) hybrid algorithm:
00383 //   ---------------------------------------------------
00384 //      init_prob_accept <float> The initial probability of accepting a
00385 //                              solution that does not improve the
00386 //                              distortion.
00387 //      temp_run_length <int>   The number of stages before chaning the
00388 //                              temperature.
00389 //      temp_reduc_factor <float> The factor by which temperature is
00390 //                              reduced at the end of a temperature run.
00391 //
00392 //  Example:
00393 //  --------
00394 //    title Experiment_1A               # experiment title
00395 //    stats summary                     # print summary information
00396 //    dim 2                             # dimension 2
00397 //
00398 //    data_size 5000                    # 5000 data points
00399 //    colors 30                         # ...broken into 30 clusters
00400 //    std_dev 0.025                     # ...each with std deviation 0.025
00401 //    distribution clus_gauss           # clustered gaussian distribution
00402 //    seed 1                            # random number seed
00403 //  gen_data_pts                        # generate the data points
00404 //  
00405 //    max_tot_stage 20 0 0 0            # terminate Lloyd's after 20 stages
00406 //  print Running-lloyd's
00407 //  run_kmeans lloyd                    # run using Lloyd's algorithm
00408 //  
00409 //    max_swaps 3                       # at most 3 swaps
00410 //    max_tot_stage 0 3 0 2             # at most 3*k^2 = 1200 stages
00411 //    max_run_stage 50                  # at most 50 stages per run
00412 //    min_accum_rdl 0.02                # stop run if distortion drops 2%
00413 //  print Running-swap
00414 //  run_kmeans swap                     # run using swap heuristic

00676 //------------------------------------------------------------------------
00677 // getDirective - skip comments and read next directive
00678 //      Returns true if directive read, and false if eof seen.

00680 
00681 static bool skipComment(                        // skip any comments
00682     istream             &in)                    // input stream
00683 {
00684     char ch = 0;
00685                                                 // skip whitespace
00686     do { in.get(ch); } while (isspace(ch) && !in.eof());
00687     while (ch == '#' && !in.eof()) {            // comment?
00688                                                 // skip to end of line
00689         do { in.get(ch); } while(ch != '\n' && !in.eof());
00690                                                 // skip whitespace
00691         do { in.get(ch); } while(isspace(ch) && !in.eof());
00692     }
00693     if (in.eof()) return false;                 // end of file
00694     in.putback(ch);                             // put character back
00695     return true;
00696 }

01019 //------------------------------------------------------------------------
01020 //  validateAssignments - validate center assignments
01021 //  This procedure is given the assignments of data points to their
01022 //  closest center, and determines (through simple brute-force search)
01023 //  whether this assignment is correct.  This is used primarily for
01024 //  debugging purposes.
01025 //------------------------------------------------------------------------
01026 
01027 void validateAssignments(
01028     KMdataPtr                   data,           // the data points
01029     const KMfilterCenters&      ctrs,           // the center points
01030     KMctrIdxArray               closeCtr,       // closest centers
01031     double*                     sqDist)         // squared distances
01032 {
01033     int errCt = 0;                              // number of errors
01034     KMdataArray dataPts = data->getPts();       // get points and centers
01035     KMcenterArray ctrPts = ctrs.getCtrPts();
01036     int nPts = data->getNPts();
01037     int kCtrs = ctrs.getK();
01038 
01039     *kmOut << "  (Validating assignments. ";
01040 
01041     for (int i = 0; i < nPts; i++) {
01042         KMdist minDist = KM_DIST_INF;           // distance to nearest center
01043         int minK = 0;                           // index of this center
01044         KMpoint thisPt = dataPts[i];            // current data point
01045 
01046         for (int j = 0; j < kCtrs; j++) {       // compute closest candidate
01047             KMdist dist = kmDist(dim, ctrPts[j], thisPt);
01048             if (dist < minDist) {               // best so far?
01049                 minDist = dist;                 // yes, save it
01050                 minK = j;                       // ...and its index
01051             }
01052         }
01053         if (sqDist[i] > minDist + KM_ERR ||     // distance mismatch
01054             sqDist[i] < minDist - KM_ERR) {
01055             *kmOut << "\n    Mismatch: data[" << i << "] assigned to ctr["
01056                 << closeCtr[i] << "] (" << sqDist[i]
01057                 << ") rather than ctr[" << minK << "] ("
01058                 << minDist << ")";
01059             errCt++;
01060         }
01061     }
01062     *kmOut << " Found " << errCt << " mismatches.)" << endl;
01063 }

01128 //------------------------------------------------------------------------
01129 //  runKmeans - run the k-means algorithm
01130 //  This procedure is given an algorithm type, a pointer to a set of data
01131 //  points and termination object.
01132 //
01133 //  This runs the k-means algorithm.  We assume that the data point set
01134 //  has been generated (dataPts != NULL).  From these we create the
01135 //  center points (using KMfilterCenters), print a header (if
01136 //  desired), resets the performance statistics and resets the clock.
01137 //
01138 //  Based on the algorithm type we create an algorithm object of the
01139 //  appropriate type and execute it.  Afterwards we print a summary of
01140 //  the results.
01141 //------------------------------------------------------------------------
01142 
01143 static void runKmeans(KMalg alg, KMdataPtr dataPts, KMterm &term)
01144 {
01145     if (dataPts == NULL) {                      // failed to create data
01146       kmError("Data points have not been generated", KMabort);
01147     }
01148                                                 // center points
01149     KMfilterCenters ctrs(kcenters, *dataPts, damp_factor);
01150     printHeader(alg, dataPts, term);            // print header
01151 
01152     KMlocalPtr theAlg = NULL;                   // the search algorithm
01153     switch (alg) {                              // select the search algorithm
01154     case LLOYD:                                 // Lloyd's algorithm
01155         theAlg = new KMlocalLloyds(ctrs, term);
01156         break;
01157     case SWAP:                                  // the swap heuristic
01158         theAlg = new KMlocalSwap(ctrs, term, max_swaps);
01159         break;
01160     case HYBRID:                                // the hybrid algorithm
01161         theAlg = new KMlocalHybrid(ctrs, term);
01162         break;
01163     case EZ_HYBRID:                             // the EZ-hybrid algorithm
01164         theAlg = new KMlocalEZ_Hybrid(ctrs, term);
01165         break;
01166     default:
01167         kmError("Internal error: Invalid algorithm", KMabort);
01168         break;
01169     }
01170 
01171     clock_t start = clock();                    // start the clock
01172     ctrs = theAlg->execute();                   // execute the algorithm
01173     exec_time = elapsedTime(start);             // get elapsed time
01174                                                 // print summary
01175     printSummary(theAlg, dataPts, ctrs);
01176 }

01262     KMdataPtr           &dataPts,       // pointer to data points (returned)
01263     bool                new_clust)      // new cluster centers desired?
01264 {
01265     int nClus;                                  // number of clusters (ignored)
01266 
01267     if (dataPts == NULL) {                      // allocate storage for points
01268         dataPts = new KMdata(dim, data_size);   // allocate new structure

01271         dataPts->resize(dim, data_size);                // else resize
01272     }

01274

01283             kmLaplacePts(pa, data_size, dim);

01285         case CO_GAUSS:                          // correlated Gaussian

01287             break;
01288         case CO_LAPLACE:                        // correlated Laplacian
01289             kmCoLaplacePts(pa, data_size, dim, corr_coef);
01290             break;
01291         case CLUS_GAUSS:                        // clustered Gaussian
01292             kmClusGaussPts(pa, data_size, dim, n_color, new_clust,
01293                         std_dev, &clus_sep);
01294             break;
01295         case CLUS_ORTH_FLATS:                   // clustered on orthog flats
01296             kmClusOrthFlats(pa, data_size, dim, n_color, new_clust,
01297                         std_dev, max_dim);
01298             break;
01299         case CLUS_ELLIPSOIDS:                   // clustered ellipsoids
01300             kmClusEllipsoids(pa, data_size, dim, n_color, new_clust,
01301                         std_dev, std_dev_lo, std_dev_hi, max_dim);
01302             break;
01303         case MULTI_CLUS:                        // multi-sized clusters
01304             kmMultiClus(pa, data_size, dim, nClus, std_dev);
01305             break;
01306         default:
01307             kmError("INTERNAL ERROR: Unknown distribution", KMabort);
